# Go Operator Service

Go Operator Service — это микросервис, написанный на языке Go, предназначенный для параллельной обработки запросов к внешним API туристических операторов.

## О проекте

Основная особенность проекта — использование worker pool архитектуры:

- входящие задачи (запросы) помещаются в очередь,
- пул воркеров обрабатывает их конкурентно,
- результаты собираются и возвращаются в унифицированном формате.

Такой подход обеспечивает:

- высокую производительность при большом количестве запросов,
- эффективное использование ресурсов,
- масштабируемость и устойчивость сервиса.

## Режимы работы

Проект поддерживает два режима работы:

- **Тестовый режим** — запросы перенаправляются на имитационный API (адрес задаётся через `.env`),
- **Продакшн режим** — прямые GET-запросы к реальным API операторов.

## Технологии

- Go (goroutines, channels, `sync.WaitGroup`)
- Worker Pool архитектура
- Docker / Docker Compose
- REST API (Echo framework)
- Поддержка конфигурации через `.env`

## Структура проекта

- `main.go` — точка входа приложения,
- `workers/` — реализация worker pool и сбор результатов,
- `models/` — структуры запросов, ответов и билетов,
- `utils/` — вспомогательные функции и трансформация данных,
- `Dockerfile`, `docker-compose.yml` — контейнеризация и запуск.

## Запуск

1. Создайте файл `.env` с необходимыми настройками.
2. Сборка и запуск через Docker Compose:

```bash
docker compose up --build
```

> Для локальной разработки достаточно установить переменные окружения и запустить `go run main.go`.
